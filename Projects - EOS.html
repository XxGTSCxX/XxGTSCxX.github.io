PROJECT - EOS

The game is still pending review and so its link will be updated accordingly once it's available.

EOS is a 3rd year Digipen games project made with our team's custom 3D game engine named "DarkHorse". For this project, I spent the first semester as the Producer before stepping down to fully focus on the technical aspects of the project due to our shortage on programmers. As a programmer, I mainly focused on the backend programming, such as:

  • Property Reflection
  • De/Serialiser
  • 3D Assets Compiler
  • Skinned Mesh Renderer
  • Bone Animations
  • Configuration Manager
  • Prefab System

This project was challenging in more ways than one and was a huge learning experience. For the most part I will be discussing what I had implemented versus what I learnt from it and believe how I should have implemented things instead.

Property Reflection

For our property reflection, we utilised Lionant's Properties project to implement our reflection engine. It laid out majority of the ground work and I wrapped it in Reflection class such that it would readable C++ format for my fellow programmers to use.
